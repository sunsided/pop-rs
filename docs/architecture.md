# Architecture

This is the canonical design for `pop-rs`. The four-pass lifter raises Apple
II 6502 (Merlin 16+) assembly into a Rust port with a pluggable backend.

## Source of truth

The upstream source lives at `vendor/pop-apple2/`, pinned via git submodule
to a specific commit of
<https://github.com/widemeadows/Prince-of-Persia-Apple-II>. The lifter
reads from this tree and never writes to it.

Important upstream files:

| Path                                                   | Purpose                                                 |
|--------------------------------------------------------|---------------------------------------------------------|
| `01 POP Source/Source/EQ.S`                            | Address book (memory banks, jump-table slots)           |
| `01 POP Source/Source/GAMEEQ.S`                        | Game-specific addresses; `dum`/`dend` struct overlays   |
| `01 POP Source/Source/MASTER.S`                        | Boot / master jump table                                |
| `01 POP Source/Source/TOPCTRL.S`                       | Main game loop                                          |
| `01 POP Source/Source/CTRL.S`, `CTRLSUBS.S`            | Character state machine                                 |
| `01 POP Source/Source/MOVER.S`, `COLL.S`               | Physics, collision                                      |
| `01 POP Source/Source/FRAMEADV.S`, `SEQTABLE.S`, `SEQDATA.S`, `FRAMEDEF.S` | Animation                          |
| `01 POP Source/Source/GRAFIX.S`, `HIRES.S`             | Graphics (replaced by `backend::Renderer`)              |
| `01 POP Source/Source/SOUND.S`, `SOUNDNAMES.S`         | Sound (replaced by `backend::Audio`)                    |
| `01 POP Source/Levels/LEVEL0..14`                      | Binary level data                                       |
| `01 POP Source/Images/IMG.CHTAB1..7`, `IMG.BGTAB1..2`  | Binary sprite / background tables                       |

Out of scope: `02 POP Disk Routines/CP.525/`, `03 Disk Protection/`,
`04 Support/` (editor + build tools).

## The four passes

### Pass 0 — lex + parse Merlin
Tokenize the Merlin dialect; build per-file ASTs; link transitively
through `put` includes into one `ProgramAST` with a flat symbol table.
`dum`/`dend` overlay blocks become named struct layouts; top-of-file
`jmp` ladders become per-module ABIs.

### Pass 1 — mechanical lift (IR1)
One IR1 function per discovered routine. Pseudo-registers `a, x, y` and
flag pseudo-globals `c, z, n, v`. Memory accesses use named globals from
the symbol table. Branches → labelled `goto`. Indexed and indirect-indexed
addressing modes lowered explicitly; 16-bit arithmetic patterns and
self-modifying code detected and annotated for later passes.

### Pass 2 — structuring (IR2)
CFG + dominators; relooper-style reconstruction of `if`/`while`/`for`.
Compare+branch fuses into conditions; dead flag updates dropped. Tagged
16-bit pairs folded. Confirmed parallel-array groups (e.g. `mobx`, `moby`,
`mobscrn`, `mobvel`, `mobtype`, `moblevel`) fuse into arrays-of-structs.

### Pass 3 — domain abstraction (IR3)
GRAFIX / HIRES / SOUND / keyboard call sites map to `Renderer`, `Audio`,
`Input` trait calls per a policy file. Data-only `.S` files lift to typed
`static` arrays. Level and sprite binaries are extracted to
`assets/extracted/`.

### Pass 4 — Rust emission
One Rust module per upstream `.S` (snake_cased). Function names preserved.
All formerly-global state lives on `struct Game`. Generated files carry a
`// @generated by pop_lifter <git-sha>` header and a manifest linking each
emitted line back to a `vendor/pop-apple2/...:line` range.

## Verification

1. **Emulator harness** — drive `linapple` or MAME `apple2ee` headless;
   snapshot RAM at deterministic checkpoints into `tests/fixtures/`.
2. **IR1 differential** — a Python interpreter executes IR1 against a
   mocked 64K+aux memory; output must match the emulator byte-for-byte
   over the routine's footprint.
3. **IR2 differential** — same interpreter on lowered IR2 must still
   match.
4. **Rust end-to-end** — `headless` backend records every trait call; the
   Rust port runs the same input tape and must produce identical
   backend-call traces and identical `Game` field values at each
   checkpoint.
5. **Renderer snapshots** — feed known sprite indices through
   `Renderer::blit_sprite` to a pixel buffer; compare to emulator PNGs.

## Pilot slice

The combat group in `AUTO.S` — `DoStrike`, `DoBlock`, `DoTurn`,
`DoStandup`, `DoEngarde`, `DoRelBtn`, and the shared `]rts` trampoline.
Smallest set that exercises multi-entry labels, `#-1`/`#0` immediate
stores, cross-routine `jmp` (tail calls), and the macro-label trampoline.
Then expand to `rndp` (RNG + indexed lookups) and `CheckFloor` in
`CTRLSUBS.S` (real branching). Only after `CheckFloor` survives all four
passes do we scale to whole files.
